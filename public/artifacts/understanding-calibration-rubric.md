# Understanding Calibration Rubric

Rubric version: `2026-07-UC-R1`

이 루브릭은 학생의 이해를 자동 채점하기 위한 도구가 아니라, 독립 설명 수행을 사람이 일관되게 코딩하기 위한 기준이다. 플랫폼은 원자료를 저장하고, 이 루브릭 점수는 별도 수동평가 데이터로 관리한다.

## 공통 코드

| Code | 0 | 1 | 2 |
| --- | --- | --- | --- |
| `accuracy` | 핵심 오류가 다수 있음 | 일부는 정확하지만 중요한 부분이 빠지거나 불명확함 | 핵심 개념이 정확하게 설명됨 |
| `completeness` | 한 가지 요소만 단편적으로 언급함 | 일부 요소를 포함함 | 문항이 요구한 핵심 요소를 충족함 |
| `coherence` | 문장이 단편적이거나 연결이 약함 | 부분적으로 연결되어 있음 | 이유와 설명이 구조적으로 연결됨 |

## 문제별 코드

### Problem 1. 자유 설명

| Code | 0 | 1 | 2 |
| --- | --- | --- | --- |
| `concept_basic` | 정의가 없거나 핵심 오해가 있음 | 일부 정의가 있으나 초등학생에게 충분히 쉽지 않음 | 초등학생이 이해할 수 있는 쉬운 정의를 제시함 |

### Problem 2. 원리 설명

| Code | 0 | 1 | 2 |
| --- | --- | --- | --- |
| `mechanism_compare` | 정보 처리 방식의 차이가 드러나지 않음 | 차이를 일부 언급함 | 일반 컴퓨터와 양자컴퓨터의 정보 처리 차이를 명확히 비교함 |
| `mechanism_keyidea` | 핵심 원리가 빠져 있음 | 핵심 원리를 얕게 언급함 | 중첩, 동시성, 정보 표현 차이 중 핵심 아이디어를 설명함 |

### Problem 3. 오개념 수정

| Code | 0 | 1 | 2 |
| --- | --- | --- | --- |
| `misconception_boundary` | 오개념을 바로잡지 못함 | 단순히 틀렸다고 말함 | 맞는 부분과 틀린 부분을 조건과 함께 구분함 |
| `correction_quality` | 더 정확한 표현이 없음 | 부분적으로 고침 | 조건과 한계를 포함해 더 정확한 설명으로 재구성함 |

### Problem 4. 적용 판단

| Code | 0 | 1 | 2 |
| --- | --- | --- | --- |
| `application_condition` | 복잡하면 무조건 좋다는 말을 그대로 수용함 | 일부 유보를 표현함 | 도움이 되는 경우와 조건을 구분해 판단함 |
| `nonapplicability` | 부적합하거나 필요하지 않은 경우를 설명하지 않음 | 암시적으로만 언급함 | 양자컴퓨터가 꼭 더 좋지 않은 경우를 함께 설명함 |

## 코딩 원칙

- 총점보다 코드별 판단을 우선한다.
- 학생의 원응답은 덮어쓰지 않는다.
- 자동채점이나 LLM Judge 결과를 원점수로 사용하지 않는다.
- 두 명 이상이 코딩할 경우 불일치 조정 기록을 별도로 남긴다.
