# 파일럿 리허설 체크리스트

작성 기준: 2026-07-02 현재 구현 기준

이 문서는 실제 학생을 투입하기 전에 연구자가 한 번씩 재현해야 하는 최소 리허설 절차이다. 목적은 화면이 보이는지 확인하는 것이 아니라, 과제 배정, 학생 수행, AI 대화, 제출, 로그 내보내기가 연구 데이터로 다시 읽힐 수 있는지를 확인하는 것이다.

## 1. 리허설 전 고정값

리허설 전에 다음 값을 고정한다.

| 항목 | 확인값 |
| --- | --- |
| 교사 계정 | 아이디와 비밀번호가 모두 `test`인지 확인 |
| 학생 계정 | 최소 2명 이상 준비 |
| 반 | 리허설에 사용할 반 1개 이상 준비 |
| 과제 유형 | `전체 글쓰기 과정`, `이해도 보정 실험` 각각 1개 이상 |
| AI 모델 | Gemini 호출 모델과 호출량 제한 확인 |
| 내보내기 파일 | JSON, research events CSV, artifacts/measures CSV |

## 2. 교사 리허설

교사 계정으로 로그인한 뒤 다음 과정을 확인한다.

- 과제 목록에서 검색, 범주, 학년 필터가 실제 목록을 바꾸는지 확인한다.
- 새 과제를 만들고 반을 선택해 저장한다.
- 과제 미리보기에서 배정할 반을 선택한 뒤 배정한다.
- 이미 만든 과제를 수정하고 저장한다.
- 계정 관리에서 반, 학생, 교사 계정을 각각 1개씩 추가한다.
- 학생 일괄 생성으로 지정한 갯수만큼 아이디와 비밀번호가 만들어지는지 확인한다.
- 만든 반, 학생, 교사 계정을 휴지통 버튼으로 삭제할 수 있는지 확인한다.
- 학생 현황에서 과제별 배정 학생, 진행 중, 제출 완료 수가 분리되어 보이는지 확인한다.
- 로그 보기에서 문제별, 학생별 기록을 구분해 추적할 수 있는지 확인한다.

## 3. 학생 글쓰기 리허설

학생 계정으로 로그인한 뒤 다음 과정을 확인한다.

- 학생에게 배정된 과제만 보이는지 확인한다.
- 로그아웃 뒤 다른 학생으로 로그인하면 이전 학생의 채팅이 보이지 않는지 확인한다.
- 읽기 단계에서 지문과 문제를 확인하고 `과제 보기`를 열어본다.
- 개요 단계에서 중심 생각, 근거, 출처, 설명, 반론을 입력한다.
- 근거 또는 예시와 출처를 추가하고 휴지통 버튼으로 삭제한다.
- 개요 점검이 부족한 항목을 정확히 알려주는지 확인한다.
- 초안 단계에서 글을 작성하고 붙여넣기 이벤트가 기록되는지 확인한다.
- 고쳐쓰기 단계에서 스크롤, 제안 보기, 내 수정 확인, 해결 표시가 작동하는지 확인한다.
- 최종 제출 후 교사 화면에서 제출 완료로 집계되는지 확인한다.

## 4. 이해도 보정 실험 리허설

이해도 보정 실험 과제에서는 다음 8단계를 끝까지 통과한다.

1. 사전 자기평가
2. 지문 읽기
3. AI와 대화
4. 예측 자기평가
5. 독립 수행
6. 사후 자기평가
7. 대화 되돌아보기
8. 완료

각 단계에서 다음을 확인한다.

- 3단계 AI 대화는 멀티턴으로 저장된다.
- 4단계에는 대화 직후 학생이 안다고 느끼는 정도가 기록된다.
- 5단계 독립 수행에서는 AI 대화창, 출처 제안, 힌트가 보이지 않는다.
- 5단계 답안은 독립 설명, 오류 판단, 전이 문제 답변으로 분리되어 저장된다.
- 7단계에서는 이전 대화를 읽을 수 있지만 새 질문을 하지 못한다.
- 8단계 완료 후 `modules.understandingCalibration.stageRecords.completed`가 내보내기 JSON에 포함된다.

## 5. 내보내기 데이터 QA

리허설이 끝난 뒤 `/export`에서 다음 파일을 확인한다.

| 파일 | 확인 항목 |
| --- | --- |
| JSON | accounts, assignments, sessions, events, chatTurns, artifacts, measures, modules가 있는지 확인 |
| JSON | 학생/교사 비밀번호가 포함되지 않는지 확인 |
| JSON | 세션이 학생, 과제, 반과 연결되는지 확인 |
| research-events.csv | 이벤트 ID, 세션 ID, 과제 ID, 학생 ID, 이벤트 유형, requestTags, aiMode, model이 있는지 확인 |
| research-artifacts-measures.csv | 독립 수행 답안, 자기평가, 대화 되돌아보기, 수동 평가 placeholder가 있는지 확인 |
| schema/codebook | 라벨링 기준과 실제 export 필드명이 어긋나지 않는지 확인 |

## 6. 중단 기준

다음 중 하나라도 발생하면 실제 파일럿을 시작하지 않는다.

- 학생이 로그인했는데 다른 학생의 채팅이나 글이 보인다.
- 새로 배정한 과제가 학생 화면에 나타나지 않는다.
- 제출했는데 교사 화면에서 제출 완료로 집계되지 않는다.
- export JSON에서 세션, 이벤트, 산출물, 측정값 중 하나가 비어 있다.
- export 파일에 비밀번호가 포함된다.
- AI가 학생 대신 최종 답안이나 문장을 작성한다.
- 모바일 화면에서 초등학생이 다음 행동을 찾기 어렵다.
- 5단계 독립 수행 중 AI 도움을 받을 수 있다.
